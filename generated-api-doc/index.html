    CRM Order Management API¶

    Auto-generated from docs/openapi/v1/openapi.yaml on 2026-04-15 10:08 UTC. Do not edit this file directly – update the OpenAPI spec and re-run generate_docs.py.

    The CRM Order Management API enables developers to create, retrieve, update, and delete customer orders within a CRM system.

    Authentication¶

    All endpoints require an API key passed in the X-API-Key header.

    Base URL¶

    https://api.example.com/v1

    Rate Limiting¶

    Requests are limited to 100 per minute per API key. Exceeding this limit returns a 429 Too Many Requests response.

    Base URLs¶

    • https://api.example.com/v1 – Production server
    • http://localhost:8000/v1 – Local development server (default: port 8000)

    Authentication¶

    Type: API Key
    Header: X-API-Key (in header)
    Description: API key for authentication. Contact support team to obtain a key.

    Example:

    curl -H "X-API-Key: your-api-key" https://api.example.com/v1/orders

    Parameter Reference¶

    The following table lists all fields used in the Order Management API.

    Parameter Type Description Required
    orderId string Unique identifier for the order. Auto-generated by the system. Yes (for specific operations)
    customerId string Unique identifier for the customer placing the order. Yes
    productName string Name of the product being ordered. Yes
    quantity integer Number of units ordered. Must be greater than zero. Yes
    price number (float) Unit price of the product Yes
    status string Current lifecycle status of the order. Yes
    createdAt string (date-time) Timestamp when the order was created. No (system-set)
    updatedAt string (date-time) Timestamp when the order was last updated. No (system-set)

    Endpoints¶

    Method Endpoint Summary
    POST /orders Create a new order
    GET /orders List all orders
    GET /orders/{orderId} Get an order by ID
    PUT /orders/{orderId} Update an order
    DELETE /orders/{orderId} Delete an order
    POST /orders/search Search orders with advanced filters
    PATCH /orders/bulk-update Update status for multiple orders

    POST /orders¶

    Summary: Create a new order

    Creates a new order in the CRM system. The orderId is auto-generated and returned in the response. The initial status is set to CREATED.
